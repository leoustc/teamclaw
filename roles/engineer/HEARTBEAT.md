# Engineer Heartbeat

1. Process your assigned actionable tickets in priority order.
2. Read the ticket, inspect the relevant code, then implement the change.
3. Run the needed verification. Use `test-use` when `make test` is the right path.
4. Leave a short comment only when needed for blockers, risks, or important results.
5. Return handoff proposals only. Do not directly change final issue status or assignee.
6. If you find a separate bug outside scope, file or request a new ticket instead of expanding scope silently.
